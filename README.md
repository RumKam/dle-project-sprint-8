# Проект: Система поиска товаров на основе дообученной CLIP модели

## Описание проекта
Разработана система поиска одежды по текстовому запросу. Пользователь вводит описание (например, *"red skirt"*), а система находит и показывает наиболее подходящие изображения товаров из базы интернет-магазина.  
Проект включает:
- Дообучение модели CLIP (Contrastive Language-Image Pre-training) на датасете одежды.
- Создание векторного индекса изображений для быстрого поиска.
- Демонстрацию работы поиска на нескольких запросах.

## Датасет
Использован датасет с Kaggle: [Fashion Product Text-Images Dataset](https://www.kaggle.com/datasets/nirmalsankalana/fashion-product-text-images-dataset).  
Содержит ~44 000 пар (изображение, описание), 142 категории одежды.

## Основные этапы выполнения

### 1. Предобработка данных
- Удаление дубликатов и пустых строк.
- Проверка наличия файлов изображений.
- Выявление и удаление плохих пар с помощью CLIP score + KMeans.

### 2. Fine‑tuning CLIP
- Модель: `openai/clip-vit-base-patch32`
- Функция потерь: контрастивная
- Оптимизатор: AdamW (lr=5e-6, weight_decay=0.01)
- Количество эпох: 5 
- **Достигнутый val CLIP score: >30** = 31.54

### 3. Система поиска
- Предвычислены эмбеддинги всех изображений (размерность 512).
- Реализована функция `search_products`, которая:
  - Вычисляет эмбеддинг текстового запроса
  - Находит top‑k ближайших по косинусному сходству
- Тестовые запросы: `"red skirt"`, `"blue sunglasses"`, `"mickey mouse"`
- Результаты визуализированы и показывают высокую релевантность.

## Результаты

### Графики обучения
- Loss монотонно снижается.
- CLIP score на валидации достигает пика на 5-й эпохе.

### Поиск
- Запрос `"red skirt"` → найдены красные юбки.  
- `"blue sunglasses"` → синие очки.  
- `"mickey mouse"` → товары с принтом Микки Мауса.  
- Время ответа < 0.02 секунды на запрос.
